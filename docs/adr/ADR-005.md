# ADR-005: Envelope encryption com seed phrase BIP-39

**Data:** 2026-03-16

**Status:** Aceita

**Autores:** Douglas Prado

---

## Contexto

Dados familiares (fotos de crianças, documentos, vídeos íntimos) são altamente sensíveis. O sistema deve garantir zero-knowledge: nem o orquestrador, nem os provedores cloud, nem os nós veem dados em texto puro. O sistema inteiro deve ser recuperável com um único segredo portável.

---

## Drivers de Decisão

- Privacidade total (zero-knowledge)
- Recuperabilidade com um único segredo (seed phrase)
- Isolamento de comprometimento (uma file key comprometida não afeta outros arquivos)
- Simplicidade para o usuário final (12 palavras, não certificados)

---

## Decisão

**Envelope encryption com hierarquia: seed (12 palavras BIP-39) → master key (PBKDF2/scrypt) → file keys (derivação) → chunk keys (AES-256-GCM)**. Vault criptografado individual por membro (cada membro tem seu próprio vault desbloqueado com sua senha) armazena tokens, credenciais e senhas. Master key existe apenas em memória e é usada para recovery do orquestrador.

**Alternativas descartadas:** Chave simétrica única (comprometimento total), cloud KMS (dependência de provedor), PGP (complexo para família não-técnica).

---

## Consequências

### Positivas

- Zero-knowledge: provedores cloud nunca veem dados em claro
- Recovery com 12 palavras: portável, memorizável, offline
- Isolamento: comprometimento de file key não afeta outros arquivos

### Negativas

- Perda da seed = perda total irreversível (sem recovery alternativo na v1)
- Complexidade de implementação da hierarquia de chaves

### Riscos

- Usuário perde seed phrase — **Mitigação:** instruções claras; alerta persistente; recovery guardians em fase futura (RF-005)
