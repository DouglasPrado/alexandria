# ADR-002: ~~Axum como web framework~~ — Substituída

**Data:** 2026-03-16

**Status:** Substituída — incorporada na ADR-001

**Autores:** Douglas Prado

---

## Contexto

Esta ADR originalmente escolhia Axum como web framework para o backend Rust. Com a migração para NestJS (TypeScript) na ADR-001, a decisão de web framework ficou incorporada na escolha do stack — NestJS já inclui o framework HTTP (baseado em Express ou Fastify), middleware, guards e interceptors como parte do core.

---

## Decisão

**Esta ADR foi substituída pela ADR-001.** O NestJS já inclui:

- **Middleware composável** — NestJS middleware, guards e interceptors substituem a necessidade de um framework separado
- **HTTP server** — built-in via Express ou Fastify adapter
- **Validação de requests** — via class-validator e class-transformer (decorators)
- **WebSocket** — @nestjs/websockets built-in

Não há necessidade de uma ADR separada para web framework quando o framework é o próprio stack.
