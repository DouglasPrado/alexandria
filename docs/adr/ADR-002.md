# ADR-002: Axum como web framework

**Data:** 2026-03-16

**Status:** Aceita

**Autores:** Douglas Prado

---

## Contexto

O orquestrador e os agentes de nó precisam de um web framework Rust para servir API REST. O framework deve suportar async nativo, middleware composável (auth, rate limiting, logging), e integrar naturalmente com o ecossistema Tokio.

---

## Drivers de Decisão

- Integração com Tokio (runtime async do projeto)
- Middleware composável para concerns transversais (auth, tracing, rate limit)
- Ergonomia da API e produtividade do desenvolvedor
- Comunidade ativa e manutenção contínua

---

## Opções Consideradas

### Opção A: Axum 0.8

Framework web async-first, built on Tokio + Tower + Hyper.

| Aspecto | Avaliação |
|---------|-----------|
| Prós | Integração nativa com Tower (middleware composável); extractors ergonômicos; Tokio-native; comunidade ativa |
| Contras | API muda entre versões major; menos features built-in que Actix |
| Esforço | Médio |
| Risco | Baixo |

### Opção B: Actix-web

Framework web pragmático e extremamente rápido.

| Aspecto | Avaliação |
|---------|-----------|
| Prós | Muito rápido em benchmarks; WebSocket nativo; streaming HTTP; HTTP/2 |
| Contras | Middleware menos composável que Tower; runtime próprio (pode conflitar com Tokio em edge cases); API menos ergonômica |
| Esforço | Médio |
| Risco | Baixo |

---

## Decisão

**Escolhemos Axum 0.8** porque a integração nativa com Tower permite compor middleware de forma natural — timeout, retry, rate limiting, tracing — que são essenciais para comunicação entre orquestrador e nós. A compatibilidade direta com SQLx (ambos Tokio-native) elimina problemas de runtime.

---

## Consequências

### Positivas

- Tower middleware reutilizável entre orquestrador e agentes de nó
- Extractors ergonômicos simplificam validação de requests
- Tracing integrado nativamente

### Negativas

- WebSocket não é built-in (precisa de axum::extract::ws, menos ergonômico que Actix)

### Riscos

- Breaking changes em versões futuras — **Mitigação:** pin de versão major; atualizar só quando necessário
