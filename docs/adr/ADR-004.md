# ADR-004: PostgreSQL 18 para metadados

**Data:** 2026-03-16

**Status:** Aceita

**Autores:** Douglas Prado

---

## Contexto

Orquestrador precisa persistir metadados: clusters, membros, nós, arquivos, manifests, chunks, réplicas, alertas. Operações críticas (criar arquivo + chunks + réplicas) devem ser atômicas. Metadata EXIF varia por arquivo (schema flexível).

---

## Drivers de Decisão

- Integridade referencial (FK entre chunks, réplicas, nós)
- Transações ACID para operações atômicas
- JSONB para metadata flexível sem migrações
- Experiência do tech lead com a tecnologia

---

## Decisão

**PostgreSQL 18 + Prisma** — type-safe queries, async nativo com Node.js, JSONB para EXIF metadata, transações ACID para consistência.

**Alternativas descartadas:** SQLite (sem concorrência), MongoDB (sem integridade referencial), CockroachDB (overkill).

---

## Consequências

### Positivas

- Queries complexas (joins chunk-replica-node) eficientes
- JSONB para metadata sem migrações por tipo de arquivo
- Ferramentas maduras de backup (pg_dump) e migração (prisma cli)

### Negativas

- Requer servidor PostgreSQL (container Docker)
- Recovery do orquestrador precisa reconstruir banco do zero (bulk insert de manifests)
